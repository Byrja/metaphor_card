# metaphor_card

Telegram-бот с авторскими метафорическими картами для саморефлексии (не гадание, не психотерапия).

## Документы
- `docs/PROJECT_DESCRIPTION_RU.md` — полное описание проекта (из ТЗ владельца)
- `docs/EXECUTION_PLAN_AGENTS.md` — пошаговый план выполнения для агентной команды
- `docs/PRODUCT_SPEC.md` — продуктовые границы MVP
- `docs/TONE_AND_SAFETY.md` — стиль и безопасность
- `docs/SESSION_FLOW.md` — сценарная драматургия
- `docs/TECH_ARCH.md` — техническая архитектура
- `docs/DATA_MODEL.md` — модель данных
- `docs/METRICS.md` — метрики продукта и пользовательская сводка

## Быстрый запуск
1. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Создать `.env` по шаблону:
   ```bash
   cp .env.example .env
   ```
3. Заполнить `BOT_TOKEN`.
4. (Опционально) заполнить `ADMIN_TELEGRAM_IDS` для доступа к `/admin_metrics`.
5. Запустить бота:
   ```bash
   PYTHONPATH=src python -m metaphor_bot
   ```

## Команды MVP
- `/start` — onboarding
- `/day_card` — карта дня
- `/check_in` — многошаговый эмоциональный чек-ин (3 шага + автосохранение)
- `/situation` — многошаговый разбор ситуации в формате 3 карт (4 шага + автосохранение)
- `/save_insight <инсайт> | <шаг>` — сохранить вывод в историю
- `/history` — посмотреть последние сохранённые инсайты
- `/patterns` — мягкий обзор повторяющихся тем по сохранённым инсайтам
- `/metrics` — персональная сводка активности (сессии/инсайты/safety)
- `/admin_metrics [days]` — агрегированные метрики по всем пользователям (только для админов), опционально за период
- `/cancel` — остановить текущую активную сессию


## Локальная проверка
```bash
PYTHONPATH=src python -m unittest discover -s tests
```

Состояние активных сценариев хранится в БД, поэтому после рестарта бота можно продолжить текущий flow.
