# QA_CHECKLIST.md

## 1) Tone & Safety (обязательно)
- [ ] Нет авторитарных трактовок смысла карты (бот не «объясняет истину за пользователя»).
- [ ] Нет пугающих, стыдящих, токсичных или манипулятивных формулировок.
- [ ] Нет диагнозов и медицинских назначений.
- [ ] Есть безопасные выходы из сессии: пауза / завершение / без углубления.
- [ ] Crisis-ответ срабатывает на red flags (high risk) и отключает игровой режим.

## 2) Core flows
- [ ] `/start` корректно показывает рамку продукта и команды.
- [ ] `/day` работает стабильно, возвращает карту и вопрос.
- [ ] `/checkin` возвращает короткий структурированный чек-ин.
- [ ] `/situation` возвращает 3-карточный сценарий + углубляющий вопрос.
- [ ] `/insight` сохраняет инсайт в БД и подтверждает сохранение.
- [ ] `/history` показывает последние записи пользователя.
- [ ] `/patterns` показывает повторяющиеся темы (если данных достаточно).
- [ ] `/nudge` возвращает мягкую подсказку по top-pattern.

## 3) Data integrity
- [ ] При старте приложения схема БД инициализируется без ошибок.
- [ ] Пользователь upsert-ится корректно по `telegram_id`.
- [ ] Сессия создаётся/завершается с корректным статусом.
- [ ] Safety-события логируются с `risk_level` и payload.
- [ ] Паттерны пересчитываются после сохранения инсайта.

## 4) Content quality
- [ ] YAML-файлы в `content/` валидны и читаются рантаймом.
- [ ] `base_mvp` содержит >= 3 карт для 3-карточного сценария.
- [ ] Prompt layers L1/L2/L3/L4 не пустые.
- [ ] Нет конфликтов с crisis-mode ограничениями (в правилах контента).

## 5) UX quality
- [ ] Сообщения короткие и читабельные (без перегрузки полотнами).
- [ ] После ключевых сценариев пользователь понимает следующий шаг.
- [ ] В истории и паттернах нет утечки данных других пользователей.
- [ ] Ошибки/пустые состояния объясняются простым языком.

## 6) Release gate (минимум)
- [ ] `py_compile` проходит на всех модулях `app/`.
- [ ] Smoke тест БД проходит (create user/session, save insight, read history/patterns).
- [ ] Smoke тест safety проходит (medium/high триггеры).
- [ ] Все изменения задокументированы в README/плане.
