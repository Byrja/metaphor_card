# Task Split: Codex + Yandex (Metaphor Card)

Цель: объединить сильные стороны двух агентов.
- Yandex: UX/тон/контент/кнопки
- Codex: стабильность, архитектура, тесты, эксплуатация

## Общие правила
1. Один PR = одна задача.
2. Каждый PR с proof: что сделано, как проверено, риски.
3. Никаких breaking changes без migration note.

---

## Зона Codex (репо GitHub: `Byrja/metaphor_card`)

### C1. Runtime reliability baseline
- Гарантировать стабильный запуск в polling (single-instance guard).
- Добавить lockfile/pid guard или явный health-check на второй инстанс.
- Логирование ошибок с понятными причинами и кодами.

**DoD:** бот работает 10+ минут без критических ошибок, нет 409 conflict.

### C2. Config & env hardening
- Чёткая валидация env (BOT_TOKEN, пути, режимы).
- `.env.example` с комментариями (dev/prod).
- Fallback-поведение при частичной недоступности зависимостей.

**DoD:** запуск в dev по инструкции с нуля за 2-3 команды.

### C3. Data/storage robustness
- Проверка инициализации схемы/хранилища.
- Защита от падений при ошибках записи (graceful UX).
- Резервный режим без БД (ограниченный, но рабочий).

**DoD:** при недоступном storage бот не умирает, а сообщает о лимитах и продолжает core flow.

### C4. Test suite uplift
- Тесты на критические флоу: `/start`, карта дня, чек-ин, завершение сессии.
- Тесты на safety guard.
- Smoke script для CI/local.

**DoD:** тесты зелёные + smoke script проходит.

### C5. Release/ops
- systemd unit + restart policy + лог ротация.
- Инструкция деплоя в `docs/DEPLOY.md`.

**DoD:** воспроизводимый запуск как сервиса.

## Зона Yandex (репо SourceCraft)

### Y1. UX polish pack
- Главное меню и кнопки (визуально/структурно) как основной сценарий входа.
- Микрокопирайтинг в стиле проекта (тепло, не слащаво, без эзотерического кринжа).
- Сократить командность интерфейса: пользовательский путь через кнопки и короткие шаги.

### Y2. Conversation design
- Проработать диалоговую драматургию по слоям (L1-L4): наблюдение → связь с собой → углубление → выход.
- Подготовить наборы формулировок для разных эмоциональных состояний.

### Y3. Content quality
- Редактура колод/подсказок/подводок: единый стиль, без противоречий.
- Удалить/исправить спорные формулировки и “ломающие атмосферу” фразы.

### Y4. Safety wording UX
- Мягкие безопасные ответы на красные флаги (без холодного тона).
- Чёткие переходы в «режим поддержки» без давления.

### Y5. Visual UX spec
- Док `UX_TEXT_GUIDE.md` + `BUTTON_MAP.md` + `SESSION_COPYBOOK.md`.
- Готовый пакет текстов для импорта в production-ветку.

---

## Интеграция (merge strategy)

1. Codex — source of truth по runtime/ops/tests.
2. Yandex — source of truth по UX-текстам/кнопкам/сценарным формулировкам.
3. Интеграционный PR в GitHub:
   - перенос UX-пакета из SourceCraft,
   - регресс-тесты,
   - smoke прогон.

---

## Acceptance критерии финала
- Бот стабилен в прод-режиме (без crash-loop, без 409 conflict).
- UX остаётся «приятным» (кнопочный вход + стильная подача).
- Сессия всегда заканчивается понятным итогом и маленьким шагом.
- Safety работает и не ломает тон проекта.
- Есть эксплуатационная инструкция и тестовый baseline.
