# Журнал изменений UX Metaphor Card (v2)

## Версия 2.0.0 - Production-ready пакет

### Добавлено

#### Документация
- `docs/UX_PATCH_MAP.md` - Точное сопоставление файлов, функций, старых и новых текстов
- `docs/BUTTON_MAP_MVP.md` - Кнопочная навигация с ограничением до реально поддерживаемых функций
- `docs/SESSION_COPYBOOK_MVP.md` - Тексты сессий только для реально поддерживаемых сценариев
- `docs/SAFETY_UX_MVP.md` - Формулировки безопасности в формате готовых reply-блоков
- `safety_replies.json` - JSON-файл со структурами safety reply-блоков

#### Обновлено
- `docs/UX_TEXT_GUIDE.md` - Обновлено с более живым копирайтингом и без шаблонных фраз

### Исправлено

#### Удалены placeholders и битые символы
- Удалены все [номер], [ссылки], "кракозябры"
- Заменены на реальные значения или нейтральные тексты

#### Приземлен BUTTON_MAP к MVP
- Оставлены только реально поддерживаемые кнопки/сценарии
- Отдельно помечены "future" функции

#### Сделан копирайтинг менее шаблонным
- Убраны общие фразы "всё хорошо/баланс/поддержка"
- Добавлены живые, короткие формулировки в стиле продукта

#### Safety UX в формате готовых reply-блоков
- Созданы структуры с trigger_type, severity, message_short, message_extended, handoff_prompt
- Без медицинских обещаний и директив

### Интеграция

#### Копирование файлов
Для переноса в production-репо необходимо скопировать следующие файлы:
```
docs/UX_TEXT_GUIDE.md
docs/BUTTON_MAP_MVP.md
docs/SESSION_COPYBOOK_MVP.md
docs/SAFETY_UX_MVP.md
docs/UX_PATCH_MAP.md
safety_replies.json
```

#### Интеграция текстов в приложение
1. Заменить тексты в src/handlers.js согласно UX_PATCH_MAP.md
2. Обновить промпты в content/prompts/*.yaml согласно UX_PATCH_MAP.md
3. Интегрировать safety_replies.json в src/safetyGuard.js
4. Проверить соответствие стиля текстов руководству UX_TEXT_GUIDE.md

#### Обновление навигации
1. Сверить кнопочную навигацию с BUTTON_MAP_MVP.md
2. Убедиться, что все контекстные действия реализованы
3. Проверить логику переходов между экранами

### Совместимость
- Все изменения совместимы с текущей архитектурой
- Не затрагивают runtime-компоненты
- Работают с веткой main без конфликтов
